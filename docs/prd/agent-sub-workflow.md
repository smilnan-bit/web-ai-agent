## 三、需求背景

**客户**：孩子王

**场景**：孩子王有 20+ 工作流，且这些工作流分别给了多个人共同维护，需要有真正的引用子工作流能力，支持员工 A/B/C 分别创建工作流 a/b/c 后，合成到一个大工作流中。而 a/b/c 支持员工 A/B/C 各自维护，并在大工作流中生效。

---

## 五、需求描述

### 1. 新增【工作流】节点

- 在【新画布】中新增【工作流】节点，支持通过该节点引用子工作流
- 节点位置放在节点列表**最上方**
- 请同步将【条件分支】节点的名称更改为【**条件判断**】

---

### 2. 选择【工作流】

- 插入【工作流】节点后，弹出弹窗，支持选择需要引用的【子流程】

#### 弹窗规格

| 字段       | 说明                                |
| ---------- | ----------------------------------- |
| 弹窗标题   | 选择工作流                          |
| 工作流信息 | `{{工作流名称}}` + `{{工作流描述}}` |
| 操作按钮   | 添加                                |

- 点击【添加】后，Toast 提示"**添加成功**"，并在画布中新增【工作流】节点

#### ⚠️ 过滤规则（重要）

> 以下三类工作流需要从列表中过滤掉：

1. **过滤引用了"自己"的工作流**（防止循环引用）——需溯源引用关系
2. **限制最大嵌套层级为 3**（包括主流程，总深度为 4）
3. **过滤掉【老工作流】**（仅展示新版工作流）

---

### 3. 设置【工作流节点】表单

#### 表单字段说明

**模块：工作流**

| 设置项              | 需求描述                                                                                           |
| ------------------- | -------------------------------------------------------------------------------------------------- |
| 工作流（出口 icon） | 点击【出口 icon】，新开页面跳转至对应【工作流详情页】，支持查看/编辑                               |
| 工作流名称          | 展示所选工作流的**最新名称**；组件置灰，不支持切换/编辑                                            |
| 工作流描述          | 展示所选工作流的**最新描述**；组件置灰，不支持切换/编辑；⚠️ 需支持通过拖拽右下角 icon 拉伸组件高度 |

**模块：输入**

| 设置项   | 需求描述                                                                                                                                                             |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 变量名称 | 展示所引用【工作流-开始节点】下所有输入参数；⚠️ 每次展开表单，均需展示**最新发布版本**的输入参数；不支持新增参数；仅展示，不支持编辑                                 |
| 变量值   | 默认展示所引用【工作流-开始节点】下的所有参数；支持重新编辑【引用/输入】；若采用引用方式，三级组件下拉列表展示当前节点**前序节点的输出参数**，需 by 参数类型进行过滤 |

**模块：输出**

| 设置项   | 需求描述                                                                                                                                                                          |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 参数名称 | 展示所引用【工作流-结束节点】下所有输出参数；⚠️ 若结束节点为【静默结束】方式，则输出参数**显示为空**；⚠️ 每次展开表单，均展示**最新发布版本**；不支持新增参数；仅展示，不支持编辑 |
| 参数类型 | 同上，若结束节点为【静默结束】则显示为空；仅展示，不支持编辑                                                                                                                      |

**模块：异常处理**

| 设置项       | 需求描述                                                                    |
| ------------ | --------------------------------------------------------------------------- |
| 异常处理方式 | 下拉单选，选项从上至下：① 中断流程（默认选中）② 返回特定内容 ③ 执行异常流程 |

交互逻辑参考【代码节点-异常处理】：

- **中断流程**：直接中断，无额外配置
- **返回特定内容**：JSON 中的参数名称需与【工作流节点-输出变量-参数名称】联动
- **执行异常流程**：在【工作流节点卡片】中展示【异常分支连接桩】

---

### 4. 【工作流节点】卡片展示

#### 卡片内容规格

| 字段       | 说明                                                        |
| ---------- | ----------------------------------------------------------- |
| 卡片名称   | 默认为"工作流"，支持重命名；⚠️ **重命名不改变原工作流名称** |
| 工作流名称 | `{{工作流名称}}`                                            |
| 工作流描述 | `{{工作流描述}}`                                            |
| 输入       | 参数名称 + 参数类型                                         |
| 输出       | 参数名称 + 参数类型                                         |
| 异常处理   | 中断流程 / 返回特定内容 / 执行异常流程                      |

#### 支持能力

- ✅ 支持【节点复制】
- ✅ 支持【节点重命名】（重命名不改变原工作流名称）

---

### 5. 【工作流节点】发布校验

点击工作流中的【发布】按钮后，需立即对【工作流】节点进行校验：

#### 校验规则

> 参考交互图中的校验项（见原文配图）

- ⚠️ 需要支持在【工作流】节点后面跟随**任意类型的节点**
- ⚠️ 当工作流流转到该节点后，需要**立即执行该画布流程**

### 9. 删除子工作流

删除工作流时，需要校验当前工作流是否被引用：

| 场景           | 处理方式               |
| -------------- | ---------------------- |
| 被引用数量 = 0 | 当前线上弹窗，无需改变 |
| 被引用次数 > 0 | 弹窗提示，阻止删除     |

#### 被引用时弹窗文案

> 该流程正在被其他工作流引用，请在下列工作流中解除引用关系后再删除。
>
> 1. `{{工作流名称1}}`
> 2. `{{工作流名称2}}`
> 3. `{{工作流名称3}}`

### 10. 子工作流更新规则

> ⚠️ 子流程**更新**后，需要按照**最新发布版本**被执行

**示例**：

若"工作流 A"同时被"工作流 B"和"工作流 C"引用：

- 当"工作流 A"修改并发布新版本后
- "工作流 B"和"工作流 C"执行时，"工作流 A"需按**最新发布版本**执行
