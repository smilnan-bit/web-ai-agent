## ADDED Requirements

### Requirement: 子工作流信息模块（只读）

工作流节点表单 SHALL 展示所引用子工作流的名称和描述（均为只读，组件置灰），并提供出口 icon，点击后新开标签页跳转到该工作流详情页。

#### Scenario: 展示子工作流名称和描述

- **WHEN** 用户展开工作流节点的右侧表单
- **THEN** 表单顶部 SHALL 展示子工作流的最新名称和最新描述，输入组件处于禁用/置灰状态，不可编辑

#### Scenario: 点击出口 icon 跳转

- **WHEN** 用户点击工作流名称旁的出口 icon
- **THEN** 浏览器新开标签页，跳转到该工作流的详情/编辑页

#### Scenario: 描述组件支持拖拽调整高度

- **WHEN** 用户拖拽描述文本框右下角的 resize icon
- **THEN** 描述文本框高度随拖拽动态变化

### Requirement: 输入参数模块

表单输入模块 SHALL 展示所引用工作流 Start 节点的所有输入参数，每次展开表单时实时拉取最新发布版本；用户可为每个参数配置引用或固定值，但不能新增或删除参数。

#### Scenario: 展示最新输入参数列表

- **WHEN** 用户展开工作流节点表单
- **THEN** 输入模块 SHALL 展示子工作流最新发布版本的 Start 节点参数列表（名称 + 类型）

#### Scenario: 配置参数为引用模式

- **WHEN** 用户将某输入参数的值模式切换为「引用」
- **THEN** 展示三级下拉组件，列出当前节点前序节点的输出参数，按参数类型过滤后可选择

#### Scenario: 配置参数为固定值模式

- **WHEN** 用户将某输入参数的值模式切换为「输入」
- **THEN** 展示输入框，用户可直接填写固定值

#### Scenario: 不允许新增参数

- **WHEN** 用户查看输入参数列表
- **THEN** 列表中 MUST 不出现「新增参数」按钮，参数数量与子工作流定义保持一致

### Requirement: 输出参数模块

表单输出模块 SHALL 展示所引用工作流 End 节点的所有输出参数（名称 + 类型），为只读展示；每次展开表单时实时拉取最新发布版本；若 End 节点为静默结束，则输出参数显示为空。

#### Scenario: 展示最新输出参数列表

- **WHEN** 用户展开工作流节点表单
- **THEN** 输出模块 SHALL 展示子工作流最新发布版本的 End 节点参数列表（名称 + 类型），组件置灰不可编辑

#### Scenario: 静默结束时输出为空

- **WHEN** 所引用工作流的 End 节点配置为「静默结束」
- **THEN** 输出参数列表 SHALL 显示为空

### Requirement: 异常处理模块

表单异常处理模块 SHALL 提供下拉单选，默认选中「中断流程」，可选项为：中断流程 / 返回特定内容 / 执行异常流程。

#### Scenario: 默认选中中断流程

- **WHEN** 用户首次展开新建的工作流节点表单
- **THEN** 异常处理方式 SHALL 默认选中「中断流程」

#### Scenario: 选择返回特定内容

- **WHEN** 用户将异常处理方式改为「返回特定内容」
- **THEN** 表单展示 JSON 配置区域，JSON 中的参数名称与输出模块的输出参数名称联动

#### Scenario: 选择执行异常流程

- **WHEN** 用户将异常处理方式改为「执行异常流程」
- **THEN** 工作流节点卡片上展示「异常」连接桩，供用户连接异常处理分支
