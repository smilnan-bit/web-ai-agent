## ADDED Requirements

### Requirement: 工作流节点必须已选择子工作流

工作流节点在发布时 SHALL 校验 `workflowId` 不为空，否则阻止发布并给出提示。

#### Scenario: 未选择子工作流时阻止发布

- **WHEN** 用户点击发布，画布中存在未配置 `workflowId` 的工作流节点
- **THEN** 发布 MUST 被阻止，并在节点上标注校验错误「请选择子工作流」

### Requirement: 输入参数映射必须完整

工作流节点的所有输入参数 SHALL 完成映射（引用或固定值），发布时 SHALL 校验所有参数均已配置。

#### Scenario: 存在未配置的输入参数时阻止发布

- **WHEN** 用户点击发布，工作流节点存在未配置值的输入参数
- **THEN** 发布 MUST 被阻止，并提示「工作流节点「{节点名}」存在未配置的输入参数」

### Requirement: 引用参数须仍存在于子工作流最新版本

发布时 SHALL 校验输入参数映射中的引用路径在子工作流最新发布版本中仍然存在，若子工作流已删除该参数则阻止发布。

#### Scenario: 引用参数已被删除时阻止发布

- **WHEN** 子工作流更新后删除了某输出参数，引用该参数的主流程尝试发布
- **THEN** 发布 MUST 被阻止，并提示「工作流节点「{节点名}」引用的参数「{参数名}」在子工作流中已不存在，请重新配置」

### Requirement: 工作流节点后可连接任意节点类型

发布校验 SHALL 允许工作流节点的输出连接桩连接任意类型的后续节点，不限制节点类型。

#### Scenario: 工作流节点后接 LLM 节点

- **WHEN** 工作流节点的输出连接到 LLM 节点
- **THEN** 发布校验 SHALL 通过，不报告连接类型错误
